# 范润然自选素材诊断训练工作台

> 版本：2026-06-11
> 对象：范润然，五年级
> 用途：你自选素材后，在本阅读诊断系统下完成“出题人加工 -> 助学者执行 -> 证据回收”的训练闭环。
> 原则：素材由你选择；题目、支架、评分和观察必须服从八轴诊断系统，不做泛泛阅读练习。
> 标准产出：一共十篇阅读题成品。每篇先判断素材可用度，再形成两个逻辑页面：学生做题端口 + 助学者/批改后解读页。

## 1. 系统定位

范润然当前不是“读不懂文章”，而是需要在中高阶阅读任务里稳定做到：

1. A4 篇章整合与概括：把零散信息压成主线、因果链、变化链。
2. A6 证据整合与论证支持：区分核心证据和背景证据，并写出“它证明了什么”。
3. A7 评价判断与迁移：观点和生活迁移都要有文本依据，不能只写常识。
4. A3 信息定位：表格、路线、数据、条件题要先看条件，再定位行列。

每次训练只抓 1-2 个主目标，不要一篇文章什么都练。

这套工作台必须在诊断系统下运行：

```text
自选素材
-> 出题人视角：提取原文证据、设计题目、写评分点、标注能力轴
-> 助学者视角：选择支架、观察作答、判断是否撤梯
-> 学生作答证据
-> 回到 A4/A6/A7/A3 的局部进步记录
```

两条红线：

- 出题人不能凭感觉出题：每道题都要绑定原文证据点和八轴能力目标。
- 助学者不能只讲答案：必须观察孩子在哪一步卡住，并记录为下一轮训练依据。

### 1.1 十篇阅读题生产协议

这里最终产出的不是素材库，也不是泛泛练习说明，而是一套 **10 篇阅读题成品包**。每一篇都必须先完成系统分析，再生成题目。

每篇强制生成链：

```text
范润然画像与本轮卡点
-> 素材类型、阅读难度、信息密度诊断
-> A1-A8 八轴素材可用度分析
-> 主训练轴与辅助观察轴选择
-> 材料证据
-> 题型识别
-> 原文可考点信号
-> 能力轴映射
-> 小题组脚手架
-> 答题公式与评分点
-> 画像回流
-> 学生做题端口：阅读材料 + 4 道主题题 + 1 道 A8 复盘题
-> 助学者解读页：答案要点 + 评分点 + 卡点提示 + 画像回流
```

本工作台默认调用两类阅读题目技能规则：

1. `eight-axis-reading-item-designer`：把材料转成“题型识别 -> 原文可考点信号 -> 可考点卡片 -> 小题组脚手架 -> 答题公式与评分点 -> 八轴画像回流 -> 系统规则”。
2. `reading-comprehension-item-techniques`：把“材料证据 -> 题型识别 -> 原文可考点信号 -> 能力轴映射 -> 小题组脚手架 -> 答题公式 -> 评分点 -> 画像回流 -> 系统规则”落实为可评分题目。

硬性要求：

- 不能把 A1-A8 直接当作题型。八轴只用于诊断、选主轴、回收画像。
- 不能看到好句子就出题。句子必须能形成可观察作答，才能成为可考点。
- 不能生成宽泛开放题。开放题也必须绑定原文句子、事件链、观点边界或证据链。
- 不能只写题目。每道题必须有原文依据、满分要点、常见低阶答案、助学者提示、评分点和画像回收。
- 不能把学生作答页和助学者解读页混在一起。学生端只放材料、题目、作答工具；解读端才放答案、评分点和诊断解释。
- 如果素材证据不足，要明确判为“谨慎使用”或“不建议作为本轮主训练素材”，不要硬凑一篇。

## 2. 标准产出：素材可用度 + 两个逻辑页面

你把素材发来后，不直接“套题”。标准产出分四层：

1. 八轴素材可用度分析。
2. 本篇综合可用度结论。
3. 学生做题端口。
4. 助学者/批改后解读页。

八轴分析是给出题人和助学者看的后台分析；学生端只看到阅读材料、题目、必要作答提示和复盘问题，不提前看到答案、评分点和诊断解释。

### 2.1 输出 A：八轴素材可用度分析

先判断这篇素材能不能承担训练任务。每一轴都要过一遍，但不等于每一轴都要出题。

| 能力轴 | 可用度 | 原文证据 | 可出题信号 | 对范润然的意义 |
| --- | --- | --- | --- | --- |
| A1 基础语言 | 高 / 中 / 低 / 不建议 | 关键词、句式、修辞、易错词 | 能否解释词句、理解指代、读懂关键句 | 只作为入口，不把本轮变成字词训练 |
| A2 文本类型与文体识别 | 高 / 中 / 低 / 不建议 | 标题、体裁特征、材料来源 | 能否判断故事、说明、规则、图表等文本类型 | 帮助她先选阅读方法 |
| A3 信息定位 | 高 / 中 / 低 / 不建议 | 显性信息、条件、表格、路线、备注 | 能否按题干条件精准定位 | 用来观察是否漏条件、看错行列 |
| A4 篇章整合与概括 | 高 / 中 / 低 / 不建议 | 时间线、原因链、变化链、段落关系 | 能否压缩为主线、因果链或变化链 | 这是本阶段主目标之一 |
| A5 推论与人物理解 | 高 / 中 / 低 / 不建议 | 行动、语言、心理、前后对比 | 能否从行为推出特点或原因 | 适合作为人物类文章的辅助轴 |
| A6 证据整合与论证支持 | 高 / 中 / 低 / 不建议 | 核心句、背景句、对比句、多处证据 | 能否区分核心证据和背景证据，并解释证据作用 | 这是本阶段主目标之一 |
| A7 评价判断与迁移 | 高 / 中 / 低 / 不建议 | 观点边界、做法、价值判断、生活连接 | 能否有文本依据地评价和迁移 | 防止只写常识或空泛感想 |
| A8 元认知监控与策略 | 高 / 中 / 低 / 不建议 | 题干误读点、证据选择点、复盘空间 | 能否说清自己怎么找、哪里卡、如何修正 | 用作训练后的过程证据 |

可用度判断口径：

- 高：原文证据清楚，能形成可观察作答，适合作为本轮训练主轴。
- 中：可以出题，但需要支架或只适合作辅助轴。
- 低：素材里有一点信号，但不稳定，不建议承担主训练。
- 不建议：缺少原文证据，容易变成常识问答或空泛表达。

### 2.2 输出 B：素材综合可用度结论

每篇素材必须给一个明确结论：

```text
综合结论：推荐 / 谨慎使用 / 不建议作为本轮主训练素材
理由：
本篇主目标轴：
本篇辅助观察轴：
不建议强行训练的轴：
阅读难度判断：偏易 / 合适 / 偏难
预计最容易暴露的卡点：
是否适合进入本轮训练：
```

如果一篇素材语言很美，但没有清楚的证据链，就可以判为“谨慎使用”或“不建议作为本轮主训练素材”。不要为了用素材而硬出题。

### 2.3 输出 C：学生做题端口

学生端仿照原“飓风之国”阅读页面：左侧阅读材料，右侧作答区，保留计时、自动保存、导出画像输入、导出作答文本、打印、清空和阅读材料字号控制。

推荐题组为“4 道主题 + 1 个 A8 复盘题”。题目顺序从低到高，保证孩子能进入，也能暴露卡点。

| 题号 | 题目功能 | 主要能力轴 | 必须绑定的原文证据 | 学生可观察输出 |
| --- | --- | --- | --- | --- |
| 1 | 定位/入口题 | A3，可兼 A1/A2 | 显性信息、题干条件、文本类型线索 | 找到信息并说出依据 |
| 2 | 主线/因果/变化题 | A4 | 时间线、原因链、段落关系 | 写出“起点 -> 变化 -> 结果”或“原因 -> 机制 -> 结果” |
| 3 | 核心证据题 | A6 | 两处以上能支持观点的原文证据 | 说明哪条证据更核心，以及它证明了什么 |
| 4 | 评价/迁移题 | A7，可兼 A5/A6 | 文中观点、人物做法、结论边界 | 有文本依据地表达判断或建议 |
| 5 | 复盘题，不计主分 | A8 | 本次作答过程 | 说清自己如何找证据、哪里卡、下次怎么改 |

每道题都必须同时写出：

```text
题目：
对应能力轴：
原文依据：
满分答案要点：
常见低阶答案：
助学者提示：
评分点：
画像回收：
```

这些后台字段不直接放在学生端页面，只进入助学者解读页。

### 2.4 输出 D：助学者/批改后解读页

助学者页不是新的做题页，而是批改后解释页。它要把“答案解析、评分点、卡点提示、撤梯条件、画像回收”放在一起，帮助家长或老师判断下一步怎么扶。

```text
本篇训练主目标：
素材可用度结论：
八轴素材可用度：
原文证据池：
每题答案要点：
常见低阶答案：
先不给提示时，先观察：
如果 A3 卡住，提示：
如果 A4 卡住，提示：
如果 A6 卡住，提示：
如果 A7 卡住，提示：
如果 A8 复盘说不清，提示：
本次可撤掉的支架：
本次必须保留的支架：

```

## 3. 你选素材时看这 6 条

优先选择：

1. 长度：800-1500 字；刚开始可用 500-900 字。
2. 难度：孩子能读完，但答题时会卡在“概括、证据、迁移”上。
3. 类型：人物故事、历史人物、科学说明文、生活规则/路线/表格材料都可以。
4. 结构：最好有清楚的时间线、原因链、变化过程、对比关系或表格信息。
5. 证据：文中至少有 3 处可引用的关键句。
6. 价值：能引出一个明确问题，例如“这个人为什么值得信任”“这个现象为什么发生”“这个做法为什么合理”。

暂时少选：

- 纯抒情、情节很散、没有明确证据点的美文。
- 只靠背景知识才能懂的文章。
- 太长的整本书节选。
- 题目只能问字词解释，无法训练主线和证据的材料。

一句话判断：**这篇文章能不能让她练“找核心证据 + 解释证据作用”？如果不能，就先不作为本轮主训练素材。**

## 4. 双角色工作流

### 4.1 出题人视角：把素材加工成题包

出题人的任务不是“多出几道题”，而是把素材变成一份可诊断、可训练、可评分的题包。

#### Step 1 先做八轴可用度判断

在文章上方写：

```text
素材标题：
素材类型：人物故事 / 历史人物 / 科学说明 / 生活规则 / 图表材料 / 其他
综合可用度：推荐 / 谨慎使用 / 不建议作为本轮主训练素材
本篇最值得读懂的是：
本篇主要训练：
本篇辅助观察：
不建议强行训练的轴：
出题人判断：这篇文章最适合暴露的卡点是：
```

#### Step 2 建立原文证据池

出题人先标，不先给孩子看。

```text
□ 关键节点：人物做了什么 / 事情怎样变化 / 原因链的一环
△ 核心证据：能直接支持题目观点的句子
○ 背景证据：有用但不能直接证明观点的句子
× 干扰证据：看似相关，但不能回答题干
```

至少形成 4 条证据：

| 证据编号 | 原文句子/段落 | 证据类型 | 可训练轴 | 可以出什么题 |
| --- | --- | --- | --- | --- |
| E1 | | 关键节点 / 核心证据 / 背景证据 / 干扰证据 | | |
| E2 | | | | |
| E3 | | | | |
| E4 | | | | |

#### Step 3 设计小题组

每篇固定 4 道主题，再加 1 个 A8 复盘题。题目顺序必须从低到高：

| 题号 | 题目功能 | 主要观察 |
| --- | --- | --- |
| 1 | 定位题 | 是否能找到显性信息或关键条件 |
| 2 | 主线/因果题 | 是否能压缩节点关系 |
| 3 | 证据题 | 是否能区分核心证据和背景证据 |
| 4 | 迁移/评价题 | 是否能有文本依据地表达 |
| 5 | 复盘题 | 是否能说清自己的找证据过程和卡点 |

#### Step 4 写评分点

每题都要写：

```text
满分答案必须包含：
1.
2.
3.

常见低阶答案：

如果出现这种答案，说明卡在：
A3 定位 / A4 主线 / A6 证据 / A7 边界
```

### 4.2 助学者视角：把题包变成脚手架

助学者拿到题包后，不重新出题，只决定怎么扶孩子完成。

#### Step 1 先问题干

```text
题目在问什么：
我找到的证据：
这条证据是核心还是背景：
它证明了什么：
```

#### Step 2 再给支架

按卡点给支架：

| 卡点 | 助学者先问 | 不直接说什么 |
| --- | --- | --- |
| A3 定位 | 题干里限制条件是哪几个？ | 不直接告诉答案在哪一行 |
| A4 主线 | 先列 3 个节点，它们是什么关系？ | 不直接替孩子概括全文 |
| A6 证据 | 这条证据能直接证明观点吗？ | 不直接说“这条没用” |
| A7 边界 | 你的建议文中哪一句支持？ | 不只评价“想法不错” |

#### Step 3 最后写完整答案

统一句式：

```text
我认为____。
证据一是“____”，它证明了____。
证据二是“____”，它证明了____。
所以____。
```

#### Step 4 记录作答证据

```text
今天我最容易错在：没看题 / 找错证据 / 只摘抄没解释 / 迁移跑远 / 表格条件看漏
助学者观察到的卡点：
是否可以撤掉一个支架：
下一篇我要先做：
```

## 5. 四类素材怎么出题

### A. 人物故事 / 历史人物

适合训练：A4、A6、A7

出题模板：

1. 这个人物经历了哪 3 个关键阶段？请用“起点 -> 变化 -> 结果”写出来。
2. 你认为这个人物最突出的特点是什么？找两处核心证据，并说明每处证据证明了什么。
3. 下面 3 条证据里，哪一条只是背景，哪两条更能支持你的观点？为什么？
4. 如果把这个人物的做法迁移到学习或生活中，你能学到什么？必须写出文中依据。

答题公式：

```text
人物是____的人。
证据一是____，它证明____。
证据二是____，它证明____。
这个启发可以迁移到____，因为文中核心不是____，而是____。
```

观察点：

- 是否把“出身、地点、年龄”当成核心证据。
- 是否只写“很努力、很善良”，没有原文证据。
- 是否能把经历压成三段，而不是复述一长串细节。

### B. 科学说明文 / 现象解释

适合训练：A4、A6

出题模板：

1. 这个现象发生的原因链是什么？请补全：原因 -> 机制 -> 结果。
2. 文中哪一句最能解释“为什么会这样”？请抄出并说明作用。
3. 下面哪条说法超出了文章能证明的范围？为什么？
4. 如果要向低年级同学解释这个现象，你会保留哪 3 个关键词？

答题公式：

```text
这个现象不是因为____，而是因为____。
先是____，接着____，最后____。
文中“____”这句话最关键，因为它解释了____。
```

观察点：

- 是否只写结果，不写中间机制。
- 是否把例子当原因。
- 是否把“这次实验/这种条件下”说成“永远都这样”。

### C. 图表 / 路线 / 规则材料

适合训练：A3、A6、A7

出题模板：

1. 根据表格，符合“____条件”的是哪一项？
2. 正文和表格分别提供了什么证据？它们合起来说明什么？
3. 如果条件变成____，原来的做法还合适吗？请按表格规则判断。
4. 你提出一条建议，必须同时符合表格规则和文章目标。

答题公式：

```text
我先看题干条件“____”。
再看表格的____列，找到____行。
正文证据是____，表格证据是____。
所以最合理的做法是____。
```

观察点：

- 是否漏看表格外的备注。
- 是否只看一列，不看条件组合。
- 建议是否违反规则或脱离目标。

### D. 抒情散文 / 成长感悟

适合训练：A4、A6、A7

出题模板：

1. 文中事物/人物发生了什么变化？请写出变化链。
2. 哪两处细节最能表现文章核心情感？为什么？
3. 结尾句的深层含义是什么？不要只解释字面意思。
4. 你能迁移到自己的生活吗？必须说明它和文章核心是同一类问题。

答题公式：

```text
表面上写的是____，其实写的是____。
证据一“____”表现了____。
证据二“____”表现了____。
我想到自己的____，相同点是____。
```

观察点：

- 是否只写“很感人、很美”，没有细节。
- 是否能说出“表面写什么，深层写什么”。
- 生活迁移是否和文章核心同类。

可直接复制打印。每篇素材用一张，分为“出题人区”和“助学者区”。

```text
姓名：范润然        日期：

【出题人区】
```

## 7. 出题人与助学者共同评分表

每篇 12 分。

| 项目 | 分值 | 看什么 |
| --- | ---: | --- |
| 读中任务 | 3 | 能否抓住题干和 3 个关键证据 |
| 主线/因果链 | 3 | 能否保留关键节点，并说清关系 |
| 核心证据 | 3 | 能否区分核心证据和背景证据 |
| 迁移边界 | 2 | 建议/感悟是否有文本依据 |
| 复盘意识 | 1 | 能否说出自己哪里容易错 |

达标标准：

- 9 分及以上，且“核心证据”至少 2 分。
- 连续两篇达标：下一篇撤掉一部分提示。
- 低于 8 分：下一篇不要加难，继续同类型材料。

角色分工：

- 出题人判断“这题是否有效”：看题目是否绑定原文证据、是否能暴露目标轴卡点。
- 助学者判断“这次是否进步”：看孩子是否少用提示、能否主动解释证据作用。

## 8. 十次训练安排

素材你来选，顺序按能力递进。

| 次数 | 素材类型 | 主练能力 | 选材提示 |
| --- | --- | --- | --- |
| 1 | 人物故事 | A6 核心证据 | 有 2-3 个具体行为，能评价人物特点 |
| 2 | 人物故事 | A4 三段概括 | 有明显起点、变化、结果 |
| 3 | 科学说明文 | A4 因果链 | 有“原因 -> 机制 -> 结果” |
| 4 | 图表/规则 | A3 条件定位 | 有表格、路线、条件或备注 |
| 5 | 历史人物 | A6 背景 vs 核心 | 有出身背景和真正成就证据 |
| 6 | 科学实验/现象 | A7 结论边界 | 有实验条件，不能过度推论 |
| 7 | 抒情散文 | A4 情感变化链 | 有表层事件和深层情感 |
| 8 | 图表+正文 | A6 正文表格互证 | 正文和表格能互相说明 |
| 9 | 人物/成长 | A7 迁移应用 | 能迁移，但必须有文本依据 |
| 10 | 综合材料 | A4+A6+A7 | 用来复测是否能独立完成 |

## 9. 每次训练后的四句复盘

1. 今天题目最关键的词是什么？
2. 你用了哪两条证据？哪条是核心，哪条只是背景？
3. 你的答案里有没有“它证明了……”这一句？
4. 如果迁移到生活，你有没有超出文章原本说的范围？

## 10. 什么时候说明训练有效

出现下面任意两项，就说明这一轮开始起作用：

- 她能主动圈题干关键词。
- 她会说“这条只是背景，不够直接”。
- 她能把“原因 -> 机制 -> 结果”补完整。
- 表格题会先找条件，再找行列。
- 迁移题不再只写常识，而会写“文中哪里支持我这样做”。

最重要的观察不是分数，而是她是否开始问自己：

```text
这条证据真的能证明我的观点吗？
```
